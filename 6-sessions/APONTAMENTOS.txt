Embora possa armazenar dados usando cookies, alguns problemas de segurança podem surgir. 
Como os cookies são armazenados no computador do utilizador, um invasor pode modificar 
facilmente o conteúdo de um cookie para inserir dados potencialmente perigosos na sua 
aplicação.

Sempre que o browser/navegador solicita uma URL ao servidor, todos os dados do cookie 
são automaticamente enviados para o servidor dentro da solicitação. 
Isso significa que, se armazenou 5 cookies no sistema do utilizador, cada um com 4 KB de tamanho, 
o navegador carregará 20 KB de dados sempre que o utilizador visualizar uma página, 
o que pode afetar o desempenho do site.

Poderá resolver estes dois problemas usando sessões do PHP. 
Uma sessão do PHP armazena dados no servidor e não no browser do utilizador. 
Num ambiente baseado em sessões, cada utilizador é identificado por meio de um número exclusivo 
chamado identificador de sessão ou SID. 
Este ID de sessão exclusivo é usado para associar cada utilizador com suas próprias informações 
no servidor, como e-mails, postagens, etc.
